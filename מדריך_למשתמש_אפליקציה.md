# 📖 מדריך למשתמש - מערכת לימוד דיקדוק למקרא (תיקון קוראים)

ברוכים הבאים למערכת האינטראקטיבית ללימוד דקדוק יהודי מסורתי!
המערכת נבנתה לפי כללי המסורה (בפרט חמשת הכללים של ר' אליהו בחור) וללא תלות בכללי האקדמיה ללשון.

## ✅ דרישות מקדימות והפעלה:
1. ודאו שמותקן לכם במחשב [Node.js](https://nodejs.org/).
2. פתחו את תיקיית הפרויקט `dikduk-app` בשורת הפקודה (Terminal).
3. הריצו פעם אחת את הפקודה להתקנת החבילות:
   ```bash
   npm install
   ```
4. להפעלת האפליקציה, הריצו:
   ```bash
   npm run dev
   ```
5. היכנסו בדפדפן הכתובת שמופיעה על המסך (לרוב: `http://localhost:5173` או `5174`).

## 🎯 מבנה המערכת

המערכת מחולקת כרגע לשני אזורים מרכזיים המוצגים כלשוניות (Tabs) בחלק העליון:

### 1️⃣ ספר הדימויים
ספר הדימויים סוקר את הפדגוגיה החזותית עליה מבוסס הלימוד שלנו. במקום ללמוד חוקי דקדוק יבשים, אנו הופכים אותם לחיים:
* 🏠 **מלך גדול (תנועה ארוכה):** מסמל תנועות יציבות (קמץ, חולם, צירי) שיכולות לעמוד בזכות עצמן.
* 🏃 **מלך קטן (תנועה קצרה):** מסמל תנועות מהירות (פתח, סגול) שרצות קדימה וזקוקות לאות הבאה כדי להיתמך.
* 🛑 **תמרור עצור (שווא נח):** כאשר מלך קטן רץ, הוא נתקע בשווא הנח שמשמש לו כקיר או תמרור עצור.
* 🏃‍♀️ **איש רץ (שווא נע):** אות שמוכרחה להשמיע תנועה למרות שאין לה מלך.
* 🛡️ **מגן (דגש קל):** מגן קשיח לאות שבא רק מתוך שקט (מצב של שווא נח או תחילת מילה).
* 🖨️ **מכונת צילום (דגש חזק):** דגש שמופעל בסערה בעקבות מכה של המלך הקודם. מכפיל את האות.

### 2️⃣ מעבדת הדיקדוק (מנתח ה-AST)
התכונה המרכזית של המערכת! זהו אלגוריתם אוטומטי (Grammar Engine) שקורא עברית מנוקדת ומנתח אותה:
1. היכנסו למעבדת הדיקדוק.
2. בחרו באחת המילים הקיימות על המסך (למשל: *וַיְדַבְּרוּ* או *בְּרֵאשִׁית*) או **הקלידו מילה מנוקדת משלכם** לתוך התיבה.
3. המערכת תפרק את המילה לאותיות ותציג:
   * איזה תנועה יש לכל אות (האם היא מלך גדול או קטן).
   * זיהוי עמוק של השווא! (האם זהו שווא נע על פי "כלל התאומות", "אחרי מלך גדול" או "בראש המילה").
   * אילו מלכים הפעילו את מכונת הצילום (דגש חזק).
4. לצד כל אות יופיעו הדימויים המלאים שילדים (וגם מבוגרים) אוהבים - אייקונים של מכונת צילום, מגנים ותמרורים, בתוספת הסבר כתוב מדוע החוק הופעל בנקודה זו!

## 🛠 הרצת בדיקות ופיתוח
ליבת הדקדוק שוכנת בקובץ: `src/engine/grammarRules.ts`.
כדי לוודא שינויים והוספת חוקים לא תפגע במסורה, יש לנו מערכת בדיקות אוטומטיות (Vitest).

כדי להריץ בדיקות במערכת:
```bash
npm run test
```
המערכת תוודא שכל הכללים מיושמים נכון (טיוטה לחוקי הדיקדוק שנרשמו בדיקות אוטומטיות).
